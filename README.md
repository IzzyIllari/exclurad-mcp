# exclurad-mcp

An MCP server that lets an AI agent drive
[EXCLURAD](https://github.com/JeffersonLab/exclurad), the Fortran code for
QED radiative corrections to exclusive meson electroproduction. The agent
translates physics intent ("η production at CLAS12 kinematics, this W × Q²
grid") into tool calls and explains the results. Deterministic, unit-tested
Python makes every physics correctness judgement. The language model never
decides whether a number is physical.

## The physics

Exclusive electroproduction experiments (ep → e'pη, ep → e'nπ⁺) measure
cross sections and beam-spin asymmetries that must be corrected for QED
radiation from the lepton before they can be compared to hadronic-model
predictions. EXCLURAD computes those corrections exactly at O(α) using the
Bardin-Shumeiko method, with no peaking approximation, driven by MAID or
EtaMAID multipole amplitudes. The output is the correction factor
δ = σ_obs/σ_Born and the corrected asymmetries, point by point in
(W, Q², cos θ*, φ*).

The η channel is new. Its formalism, results, and CLAS12-kinematics dataset
are published in:

> I. Illari, A. Afanasev, W.J. Briscoe, V.L. Kashevarov, A. Schmidt,
> I. Strakovsky, *Amplitude-Based Analysis of QED Radiative Corrections to
> Electroproduction of η-Mesons on Protons*,
> [arXiv:2604.22943](https://arxiv.org/abs/2604.22943),
> [accepted in Phys. Rev. D](https://journals.aps.org/prd/accepted/10.1103/78q1-72dn) (2026).

Browse the computed corrections interactively at
[izzyillari.github.io/exclurad](https://izzyillari.github.io/exclurad/):
1.9 million kinematic points spanning W = 1.487–2.211 GeV and
Q² = 0.3–5.0 GeV².

## What this server has actually done

The paper's original grids were generated by hand over months: thousands of
input files against a code whose interface knowledge lived in people's
heads. This package encodes that knowledge as code and has since carried a
full production campaign on its own:

- It validated and generated the July 2026 extension campaign: 1.13 million
  new points extending the grid from W = 2.0 to 2.211 GeV, preflight-checked
  before any compute was spent, run on LANL cluster nodes in under 100
  node-hours, parsed and merged into the public dataset.
- Points that overlap the hand-validated October 2025 grids reproduce the
  published values to exact bit equality.
- Coverage mapping accounted for every point: 99.988% of the
  1.4-million-point target grid reconciled, and the remainder traced to 17
  identifiable integrator timeouts. Nothing was silently lost.
- The parsers and coverage accounting surfaced a latent bug in the 2002
  integrator: about 1% of δ values in every η grid ever produced were NaN.
  The cause is an unguarded square root evaluated at shifted kinematics a
  few parts in 10⁷ above threshold, present in the published code for two
  decades. A phase-space guard has been validated bit-for-bit against
  unaffected points and is being reviewed with the code's authors.

The deterministic layer did the catching and the proving in each case. The
agent's job is orchestration and explanation.

## Why the validation layer exists

EXCLURAD's interface has sharp edges that no model should be trusted to
remember:

- Lookup-table filenames are historical aliases. `maid07-PPpi.tbl` in the
  η configuration actually contains EtaMAID-2023 η multipoles.
- cos θ* must never be exactly ±1 (the validated convention is ±0.999).
- Input files take at most 10 points, need two specific blank lines, and end
  with an inert `0error detected by nag library routine...` trailer.
- Unphysical kinematics either hang the integrator forever or exit cleanly
  with no result, a silent N/A.

Each of these is encoded once, as tested code, in `validators.py`,
`channels.py`, and `inputgen.py`.

## Tools

| Tool | What it does |
|---|---|
| `list_channels` | Known channels with thresholds, defaults, table aliases, quirks |
| `resolve_tables` | Which `.tbl` the Fortran will open, what it really contains, and its (W, Q²) grid scanned from the file itself |
| `preflight_check` | Deterministic PASS/WARN/FAIL per kinematic point: threshold, beam accessibility, pole safety, table coverage, vcut sanity, with fix suggestions |
| `generate_input` | Byte-compatible input files (auto-chunked, manifest.csv); refuses on preflight FAIL |
| `run_exclurad` | Execute with timeout; classify OK / TIMEOUT / EXIT_NONZERO / NO_TAI (silent N/A) / NO_OUTPUT; collect the seven output files |
| `smoke_test` | Probe one point with a short timeout before committing a grid |
| `map_failures` | Aggregate skip logs or compute coverage maps by kinematic region, to diagnose why regions fail |
| `describe_outputs` | Column, unit, and separator documentation for all seven output files, decoded from the Fortran write statements |
| `parse_output` | Parse one output file or a whole results tree into a tidy table or CSV; auto-detects η vs π⁺ build formats |
| `describe_build_slots` | The 16 places where the Fortran hardcodes a channel-specific value, with accepted values per channel |
| `generate_build` | Render channel sources from the single template plus slot registry; optional byte-identity verification against a validated checkout |

## One source, sixteen slots

The η and π⁺ configurations differ in exactly 16 localized regions of the
Fortran (four meson-mass sites, polarization mode, integration tolerance,
output formats, the table-header read format, lookup-table grid dimensions,
plus cosmetics). `fint.F` and `spp.inc` are identical between them.
`templates/` holds one template per file with named `@@SLOT@@` markers and a
registry of the accepted text per channel.

The correctness contract is byte identity, verified in both directions:
generating `eta` reproduces the [IzzyIllari fork](https://github.com/IzzyIllari/exclurad)
exactly, and generating `pion` reproduces
[JeffersonLab upstream](https://github.com/JeffersonLab/exclurad) exactly.
A π⁺ executable compiled from generated sources produced byte-identical
outputs (all seven files) to an upstream-source build on upstream's own
`input.dat`. Adding a new channel (kaon, say) means adding that channel's
text to the slots that change, and the registry documents which those are.

## Output-format landmines (encoded in `outputs.py`)

- `radtot.dat` and `radsigpl.dat` are CSV; `radcor.dat`, `radasm.dat`, and
  `radsigmi.dat` are fixed-width. The formats were toggled during the code's
  development and never unified.
- `radtot` column 1 (`e1`) is never populated. The beam energy is the input
  `bmom`.
- `radcor.dat` has two rows per kinematic point (unpolarized, polarized).
- Fixed-width fields abut with no separator when full (`phi=252.00` in f6.2),
  so those files must be sliced by width. Their f6.2 kinematic columns round
  cos θ* = 0.999 to 1.00; join to `radtot` for exact point identity.

## Install and run

```bash
conda env create -f environment.yml   # python + gfortran + make
conda activate exclurad
pip install -e .
cd /path/to/exclurad/exclurad && make # builds build/exclurad

export EXCLURAD_WORK_DIR_ETA=/path/to/IzzyIllari-exclurad/exclurad
export EXCLURAD_WORK_DIR_PIPLUS=/path/to/JeffersonLab-exclurad/exclurad
exclurad-mcp
```

η and π⁺ are different builds. Channel constants and masses are compiled in
via `mpintp.inc`, so each channel points at its own checkout. Swapping
`.tbl` files alone would silently produce wrong physics.

Claude Code registration:

```bash
claude mcp add exclurad \
  -e EXCLURAD_WORK_DIR_ETA=/path/to/IzzyIllari-exclurad/exclurad \
  -e EXCLURAD_WORK_DIR_PIPLUS=/path/to/JeffersonLab-exclurad/exclurad \
  -- exclurad-mcp
```

Any MCP-speaking agent works the same way; the server makes no LLM and no
network calls of its own.

Tests: `PYTHONPATH=src python3 -m pytest tests/`

## Beyond EXCLURAD

The channel registry (`channels.py`) is the extension point for other
exclusive processes. The pattern itself, corpus-grounded preflight
validation of simulation inputs for legacy physics codes, applies to any
code with the same Fortran-era pain points: undocumented flags, hardcoded
lookup-table aliases, strict input formats, kinematic regions that hang or
silently fail.

## Corpus

`corpus/exclurad-bibliography.md` collects the methodology papers, 46
